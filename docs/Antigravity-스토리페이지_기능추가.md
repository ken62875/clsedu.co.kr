# 스토리(블로그) 목록 및 상세 페이지 기능 추가

## 과제 목표
* https://clsedu.co.kr/story 경로에 블로그 아티클 목록 페이지 구축.
* 데스크탑 환경에서 한 줄에 4개 카드 (4칼럼), 모바일 기기(스마트폰) 환경에서 한 줄에 2개 카드 (2칼럼)씩 보이도록 반응형 그리드 뷰 화면 설계.
* 한 페이지당 20개의 게시글 노출.
* 하단에 페이지네이션(Pagination) 기능 구현.
* 개별 스토리 상세 내용을 보여줄 싱글 포스트 템플릿(상세 페이지) 구축.

## 구현 상세 내역

### 1. 테스트용 목업 데이터 추가 (`src/data/mockStoryData.ts`)
* 페이지네이션 테스트 및 카드 UI 검증을 위해 45개의 목업 스토리 데이터를 생성하였습니다.
* 카테고리(학원소식, 입시칼럼, 성적향상, 공부법), 작성일자, 제목, 썸네일 경로, 요약문, 본문 HTML 등의 필드가 포함되어 있습니다.

### 2. 페이지네이션 컴포넌트 (`src/components/ui/Pagination.tsx`)
* Next.js App Router 환경에 적합하도록 정적 `Link` 버튼들을 렌더링하는 `Pagination` 컴포넌트를 구성했습니다.
* `?page=N` URL 쿼리(Search Parameters) 방식을 사용하여 별도의 클라이언트 상태(State) 없이 서버에서 페이지 값을 받을 수 있도록 구현하였습니다.

### 3. 스토리 목록 페이지 (`src/app/story/page.tsx`)
* Next.js 서버 컴포넌트(`async function StoryPage`)에서 `searchParams`를 읽어 현재 페이지의 번호를 계산합니다.
* `mockStoryData`를 이용해 전체 데이터 중 해당 페이지에 해당하는 20개의 데이터만 추출(`slice`)합니다.
* **반응형 그리드 구현**: 
  TailwindCSS의 `grid-cols-2` 속성을 통해 모바일에서 2칼럼으로 디스플레이하고, `md:grid-cols-4` 속성을 적용해 태블릿/PC(데스크탑) 화면에서는 4칼럼으로 노출되도록 조정하였습니다.
* `FadeIn` 공통 컴포넌트를 적용하여 스크롤 및 로드 시 부드럽게 카드가 표시되도록 애니메이션 효과를 접목했습니다.

### 4. 싱글 포스트(상세 페이지) 템플릿 (`src/app/story/[id]/page.tsx`)
* Next.js의 동적 라우팅(`[id]`)을 활용하여 상세 정보를 가져오도록 페이지를 구성했습니다.
* 상단에는 카테고리, 날짜, 작성자 정보를 배치하고 메인 이미지를 와이드 비율로 표시합니다.
* 상세 본문 영역을 가상으로 채워두었으며, 추후 CMS나 마크다운 파서 등을 연결할 경우를 대비하여 `dangerouslySetInnerHTML` 방식이나 `<article>` 태그의 `prose` 설정을 호환하도록 했습니다.
* 하단에는 '목록으로' 버튼과 '상담 문의하기' 링크를 배치하여 유저 경험(UX) 및 전환율 향상을 고안하였습니다.

## 확인 및 검증 방법
1. 로컬 환경에서 Next.js 서버 구동 (이후 터미널에 `npm run dev` 실행)
2. `http://localhost:3000/story`로 접속하여 페이지별 20개의 카드가 노출되는지 확인.
3. 브라우저 창 너비를 스마트폰 사이즈로 줄여서 2칼럼으로 제대로 줄어드는지 확인.
4. 하단 페이지네이션 번호를 눌러 URL 및 목록 업데이트 확인.
5. 임의의 카드를 클릭하여 템플릿 페이지(`/story/[id]`)가 정상적으로 렌더링되는지 확인.
