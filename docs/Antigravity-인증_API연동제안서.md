# app.clsedu.co.kr 외부 API 연동 및 패스워드 설정 플로우 개발 제안서

본 문서는 `app.clsedu.co.kr`의 사용자 정보를 활용한 인증 연동 구축 시, 프론트엔드(Next.js)측에서 구현한 API 라우트 프록시 설계와 프론트엔드 연동 현황을 정리한 문서입니다.

## 1. 프론트엔드 - 백엔드(app.clsedu.co.kr) 통신 규격 설계 방안

Cross-Origin 문제와 토큰 보안(보안 쿠키 관리)을 위해 서버 사이드에서 `app.clsedu.co.kr` 외부 API로 직접 HTTP 요청을 보내는 API Route(Proxy) 구조를 설계했습니다. 현 상황은 로컬 더미 지연(500ms)이 적용된 상태이며, 추후 `fetch` 주석 처리 부분을 실제 백엔드 호스트로만 교체하면 즉시 서비스가 가능하도록 준비해 두었습니다.

### 1-1. 메인 로그인 동작 (`/api/auth/login`)
- 로그인 시 `POST /api/auth/login`을 호출합니다.
- 백엔드에서 받은 인증 정보를 기반으로 Next.js 컨텍스트에 HTTP-Only 방식의 쿠키(세션)를 저장합니다.
- **예외 처리 전략**: `app.clsedu.co.kr`의 계정 중 '비밀번호가 없는 계정' 정보로 로그인을 시도했을 경우, **비밀번호 설정이 필요함을 뜻하는 특정 상태코드(예: 403)와 `PASSWORD_NOT_SET` 문자열 에러**를 내려주도록 백엔드 설정을 권장합니다.
- 프론트엔드는 이 에러 발생 시 `/login/set-password` 경로로 즉시 리다이렉트합니다.

### 1-2. 비밀번호 미설정 사용자의 초기 비밀번호 세팅 (3단계 Flow)
비밀번호 설정 화면(`src/app/login/set-password/page.tsx`)은 연락처 기반 인증 후 패스워드를 지정하는 구조이며, 다음 세 가지 라우트를 통해 연동됩니다.

1. **`POST /api/auth/send-verification`**: 입력한 이메일/휴대폰번호로 인증 코드를 전송하도록 백엔드에 요청합니다.
2. **`POST /api/auth/verify-code`**: 전송된 6자리 코드와 계정정보를 검증하고, 성공 시 비밀번호를 변경할 수 있는 **임시 토큰(`reset_token`)**을 반환하도록 백엔드에 요청합니다.
3. **`POST /api/auth/set-password`**: 전달받은 임시 토큰과 신규 비밀번호를 통해 최종적으로 사용자 비밀번호를 갱신합니다.

### 1-3. 전역 상태 연동 (`/api/auth/me`)
사용자가 사이트를 새로고침할 때 브라우저에 쿠키가 존재하면, 해당 정보를 복호화하여 브라우저의 React 클라이언트로 반환해 즉각 로그인 뷰(Header 변경 유지)를 렌더링합니다.

## 2. 작업 완료 화면 상세 (더미 테스트 방법)

지금 바로 개발 환경 브라우저에서 테스트할 수 있습니다.

- **비밀번호 있는 사용자 (일반/관리자 지정)**:
  - 아이디: `admin` / 비밀번호: `1234` 입력 시, 관리자로 마이페이지 진입 성공.
  - 아이디: `superadmin` / 비밀번호: `1234` 입력 시, 최고관리자 접속 성공.
- **비밀번호 없는 사용자 분기**:
  - 아이디: 임의 작성 아무 거나 / 비밀번호: `nopassword` 를 입력하여 로그인 버튼을 누르면 서버 측에서 "비밀번호 미설정 에러"를 시뮬레이션 하며 3단계 패스워드 재설정 창으로 화면이 자동으로 넘어갑니다.
- **초기 비밀번호 셋업 테스트**:
  - 전송된 더미 인증번호는 항상 `123456` 입니다. इसे 사용해 비밀번호 설정 완료 페이지까지 정상 동작하는지 테스트합니다.
